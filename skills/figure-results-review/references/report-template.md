# Report Template

Use this structure for a figure or result review.

```markdown
# Figure Results Review: [Project/Result]

## Review Context

- Target audience:
- Target claim:
- Figure/table inputs:
- Linked memory IDs:

## Decision Summary

- Overall readiness:
- Main claim status:
- Highest reviewer risk:
- Required fixes before paper/meeting:
- Recommended next skill:

## Figure and Table Inventory

| ID | Asset | Intended claim | Status | Paper location | Main risk |
|---|---|---|---|---|---|

## Claim Support Audit

| ID | Evidence relation | Status | Claim change needed | Reviewer question answered |
|---|---|---|---|---|

## Visual and Table Integrity

| ID | Issue | Severity | Fix |
|---|---|---|---|

## Statistical and Experimental Evidence

| ID | Missing or weak evidence | Severity | Fix or claim change |
|---|---|---|---|

## Caption and Narrative Fixes

### [ID]

- Caption diagnosis:
- Revised caption:
- Paper callout:
- Claims to avoid:

## Reviewer Risk Forecast

| Risk | Reviewer attack | Severity | Mitigation |
|---|---|---|---|

## Routed Actions

| Action | Type | Owner/next skill | Exit condition |
|---|---|---|---|

## Project Memory Update

- Evidence:
- Claims:
- Risks:
- Actions:
```

Keep the report focused on decisions. Long raw result logs belong in experiment reports.
