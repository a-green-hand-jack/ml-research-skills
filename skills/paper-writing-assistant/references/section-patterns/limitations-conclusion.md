# Limitations and Conclusion Patterns

Use this file when drafting limitations, broader impact, ethics, conclusion, and scope statements.

## Limitations Pattern

1. Scope limitation: where the evidence does not yet apply.
2. Mechanism limitation: where the proposed explanation may fail.
3. Evaluation limitation: what the benchmark, metric, data, or protocol cannot measure.
4. Practical limitation: compute, data, deployment, usability, or safety constraint.
5. Future work: only if it follows from the limitation.

Good limitation language:

- "Our evidence supports [claim] in [setting], but does not establish [broader claim]."
- "This result should be interpreted as [narrow claim], because [missing evidence or protocol constraint]."
- "The current benchmark isolates [capability], but does not measure [adjacent capability]."

Avoid:

- generic limitations that could apply to any paper
- limitations that introduce new unsupported claims
- apologizing for choices that are actually design boundaries

## Conclusion Pattern

1. Restate the problem or gap in one sentence.
2. State the contribution in claim-aware terms.
3. Summarize the strongest evidence.
4. State the implication or enabled future work.
5. Avoid adding a new claim that was not supported in the paper.

Avoid:

- repeating the abstract verbatim
- ending with hype
- listing every experiment again
