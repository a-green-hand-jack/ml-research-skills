# Report Template

Use this structure for a baseline selection audit.

```markdown
# Baseline Selection Audit: [Project/Claim]

## Claim Under Audit

## Decision Summary

- Baseline readiness:
- Main missing risk:
- Must-have baselines:
- Should-have baselines:
- Baselines to exclude:
- Recommended next skill:

## Candidate Baseline Pool

| Candidate | Role | Requirement | Why reviewers expect it | Comparison form | Source/status |
|---|---|---|---|---|---|

## Selected Baseline Set

| Baseline | Main/appendix | Metric/protocol | Fairness constraints | Reviewer question answered |
|---|---|---|---|---|

## Fairness Ledger

| Baseline | Data | Capacity | Compute | Tuning | Protocol | Metric | Implementation | Verdict |
|---|---|---|---|---|---|---|---|---|

## Tuning Budget Ledger

| Method | Search space | Trials | Selection metric | Compute budget | Notes |
|---|---|---|---|---|---|

## Reviewer Attack Forecast

| Baseline issue | Reviewer attack | Severity | Mitigation | Owner/action |
|---|---|---|---|---|

## Excluded or Citation-Only Baselines

| Baseline | Status | Reason | Text/action needed |
|---|---|---|---|

## Experiment Handoff

### Minimal Reviewer-Proof Set

### Optional Additions

### Metrics and Protocol Constraints

### Stop Conditions

## Claim Changes

## Project Memory Update

- Decisions:
- Claims:
- Risks:
- Evidence:
- Actions:
```

Keep the report short enough to guide actual experiment planning. Put long paper notes in the literature review report.
