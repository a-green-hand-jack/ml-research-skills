# Release and Artifact Handoff

Camera-ready finalization should expose release obligations clearly.

## Code Release Handoff

For `release-code`, provide:

- repository path and intended public URL
- license decision
- README status
- reproduction commands
- model/data/checkpoint links
- expected hardware and runtime
- paper citation and `CITATION.cff` needs
- release tag name
- security or secret audit status
- large-file or dataset policy

## Artifact Evaluation Handoff

For `artifact-evaluation-prep`, provide:

- install path
- minimal demo
- expected runtime
- expected outputs
- hardware assumptions
- data download or synthetic fallback
- checkpoint availability
- troubleshooting notes
- artifact badge or evaluation deadline if known

## Paper Link Handoff

Before adding links to the paper:

- confirm destination exists or will exist before upload
- confirm public/private visibility plan
- avoid linking to unstable personal paths
- ensure README and paper claims match

## Milestone Handoff

For `add-git-tag`, summarize:

- accepted paper title
- venue/year
- final camera-ready status
- release status
- remaining post-camera-ready tasks
