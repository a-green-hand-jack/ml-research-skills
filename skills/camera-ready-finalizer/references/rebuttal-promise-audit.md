# Rebuttal Promise Audit

Camera-ready edits must honor promises made during author response and discussion.

## Sources

Collect promises from:

- rebuttal text
- follow-up discussion replies
- reviewer comments after rebuttal
- meta-review or area chair note
- acceptance email
- internal issue board from `rebuttal-strategist`

## Promise Ledger

```markdown
| ID | Source | Promise / commitment | Required final change | Status | Paper location | Risk |
|---|---|---|---|---|---|---|
```

Statuses:

- `fulfilled`
- `partially-fulfilled`
- `superseded`
- `not-applicable`
- `missing`
- `needs-author-decision`

## Common Promise Types

- add experiment or ablation
- add limitation
- clarify assumption
- cite missing related work
- fix notation or proof detail
- report extra metric
- update figure/table/caption
- release code or data
- add reproducibility details
- correct typo or ambiguity

## Risk Handling

If a promise is missing:

1. decide whether it is blocking
2. identify the smallest faithful final-paper edit
3. update action board
4. mark residual risk if not fulfilled

Avoid adding new strong claims while fulfilling a promise.
