# Narrative Architecture

Convert the positioning decision into paper-level structure.

## Title Direction

A title should signal:

- contribution type
- method or object of study
- target problem or domain
- key property if distinctive

Avoid title claims that require missing evidence.

## One-Sentence Thesis

Use:

```text
We show that [core move] enables [specific benefit] for [setting], supported by [main evidence], unlike [closest work limitation].
```

## Abstract Skeleton

1. Problem or gap.
2. Closest-work limitation.
3. Proposed contribution.
4. Main evidence.
5. Scope and implication.

## Intro Paragraph Roles

```text
P1: Problem and why readers should care.
P2: Why existing work is insufficient or leaves a gap.
P3: Core insight or contribution.
P4: Evidence summary and paper shape.
P5: Contributions list with scoped claims.
```

## Main Figure or Table Role

Decide whether the main visual should be:

- method schematic
- speed-quality curve
- headline comparison table
- claim-evidence map
- diagnostic mechanism plot
- benchmark/protocol overview

The main visual should support the primary contribution, not a secondary curiosity.

## Related-Work Boundary

Specify:

- work we directly compete with
- work we build on
- work we cite for context
- work outside our claim scope
- terminology to use consistently

## Claims-to-Avoid Box

Include a short list:

```markdown
Do not claim:
- ...
```

This prevents later writing from drifting beyond evidence.
