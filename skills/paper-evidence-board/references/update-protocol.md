# Update Protocol

Use this protocol whenever the paper board changes.

## When New Results Arrive

Update:

- evidence status
- linked claims
- figure/table status
- sections that mention old results
- risks opened or closed
- actions for writing, diagnosis, or new experiments

If a result weakens a claim, mark the claim as weakened before rewriting.

## When Writing Changes

Update:

- claim wording
- paper location
- claim strength
- section map
- figure/table roles
- unsupported or newly introduced claims

Writing can create new experiment actions when it exposes evidence gaps.

## When Review Simulation Runs

Update:

- reviewer risks
- risk priority
- affected claims and sections
- actions grouped by experiment, rewrite, citation, or limitation

Do not treat simulated reviews as observed reviews.

## When Real Reviews Arrive

Update:

- reviewer issue IDs
- observed risks
- rebuttal actions
- promised revisions
- claims that need clarification or evidence

Real review issues should also be tracked by `rebuttal-strategist`.

## When Claims Are Cut

Record:

- claim ID
- old paper location
- reason for cut
- evidence or risk that caused the cut
- sections/figures/actions to clean up

Cut claims should not silently reappear in the abstract or introduction.

## Closeout

End with:

- top unsupported claims
- stale figures/tables
- blocking reviewer risks
- next 3 actions
- next skill to use
