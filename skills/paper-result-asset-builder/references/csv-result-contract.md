# CSV Result Contract

CSV files are the raw material for paper-facing evidence. Treat them as structured evidence sources.

## Recommended Columns

Prefer CSVs that contain:

- `run_id`
- `experiment_id`
- `method`
- `baseline`
- `dataset`
- `split`
- `seed`
- `metric`
- `value`
- `step` or `epoch` when curves are needed
- `config_path`
- `commit`
- `timestamp`
- `status`

Wide-format CSVs are acceptable, but record how metric columns map to paper metrics.

## Required Metadata

For paper-facing use, recover or record:

- metric direction: higher is better / lower is better
- unit
- aggregation rule
- seed/repeat count
- dataset split
- baseline fairness setting
- run status: complete, failed, partial, excluded
- source certainty: verified, user-stated, inferred, stale, needs-verification

## Inventory Rules

Before loading large CSVs into context:

- list paths
- inspect headers
- count rows
- sample a few rows
- identify candidate method/dataset/metric/seed columns
- note file modification time and rough size

Use the inventory as a map, not as final evidence.

## Selection Rules

When selecting rows for a paper asset:

- filter only by explicit columns or documented filename conventions
- keep the filtering rule reproducible
- exclude failed or partial runs only with a recorded rule
- separate main-paper evidence from diagnostic or appendix evidence
- do not average across datasets, tasks, or metrics unless the paper claim asks for that aggregate

## Aggregation Rules

For repeated runs:

- aggregate per method/dataset/metric after filtering valid runs
- report mean plus standard deviation, standard error, confidence interval, or individual seeds when needed
- avoid hiding unstable results with a single mean
- preserve sample size

For benchmark tables:

- define metric direction before bolding
- bold only after aggregation and rounding rules are fixed
- explain ties and missing values

## Provenance Status

Use:

- `verified`: CSV inspected and rules documented
- `generated`: asset generated by a script from verified CSVs
- `manual-edit`: hand adjustment exists and is documented
- `user-stated`: user provided mapping or number without source inspection
- `needs-verification`: source exists but mapping is incomplete
- `stale`: CSV or asset is older than known newer results
