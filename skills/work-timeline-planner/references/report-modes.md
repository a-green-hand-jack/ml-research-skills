# Report Modes

Use this reference to shape the final report for the user's audience.

## 1. Personal Review

Use when:

- the user wants to look back on their own time allocation
- the review spans multiple projects
- the report is mainly for internal reflection or weekly planning

Emphasize:

- project split
- workstream split
- where time went
- what advanced, stalled, or shifted

Keep:

- more project breadth
- medium technical detail

## 2. Mentor Report

Use when:

- the user is reporting one project's recent progress to a mentor or collaborator
- the report needs to justify time spent and show concrete outputs

Emphasize:

- deliverables
- outcomes
- blockers
- next step

Keep:

- one-project focus
- lower noise from tiny technical changes
- stronger explanation of impact

## 3. Next-Phase Plan

Use when:

- the user wants to plan the next stage
- the user needs a realistic sequence of tasks

Emphasize:

- milestones
- dependencies
- uncertainty
- near-term next actions

Keep:

- planned tasks separate from completed tasks
- future blocks realistic and milestone-based

## 4. Hybrid Review + Plan

Use when:

- the user wants to show what was done and what comes next

Recommended structure:

1. Scope and evidence
2. Retrospective Gantt
3. Short retrospective summary
4. Next-phase Gantt
5. Immediate next actions
